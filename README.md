# 🎤 Techview

IT 취업 준비생을 위한 기술 모의면접 서비스 (Frontend)

Techview는 IT 취업 준비생이 혼자서도 실전 같은 기술 면접을 반복 연습하고, 자신의 답변을 분석할 수 있도록 돕는 웹 서비스입니다.  
포지션·기술 스택 기반 면접 질문 선택 → 실시간 녹화 면접 → AI 피드백 확인 → 커뮤니티 공유까지,  
하나의 서비스 안에서 면접 준비 전 과정을 경험할 수 있도록 설계되었습니다.

---

## 🔍 한눈에 보기
- **타겟 사용자**: IT 취업 준비생 (프론트엔드, 백엔드 등)
- **목표**: 반복적인 기술 면접 연습 + 자기 분석 환경 제공
- **핵심 가치**
  - 실전과 유사한 녹화 기반 모의면접
  - 질문별 AI 피드백 시각화
  - 면접 경험을 공유하는 커뮤니티
  - 사용자 / 관리자 분리된 운영 구조

---

## 🖥️ 주요 화면 & 흐름
1. **메인 랜딩**
   - 서비스 태그라인 및 브랜딩
   - 문제 수, 사용자 수 등 신뢰 지표
   - 상단 공용 헤더를 통한 주요 기능 진입 (면접 · 커뮤니티 · 마이페이지 · 관리자)
2. **포지션 / 기술 스택 기반 면접 선택**
   - 직무별 카드 UI + 호버 이펙트
   - 기술 스택 및 질문 선택
   - 북마크 기능 제공
   - 선택 결과를 쿼리 파라미터로 면접 화면에 전달
3. **실시간 모의 면접**
   - 질문 타이머 + 5초 카운트다운
   - 카메라 / 마이크 권한 요청
   - MediaRecorder 기반 영상 녹화
   - 질문 전환 시 타이머 및 녹화 상태 자동 관리
4. **면접 세션 관리**
   - 면접 종료 후 세션 ID · 질문 메타데이터 · 영상 정보 localStorage 저장
   - 세션 리스트에서 썸네일, 영상 길이, 실패 사유 등 요약 정보 확인
5. **AI 피드백 화면 (Mock)**
   - 질문별 AI 분석 탭
   - 감정 변화 그래프
   - 키워드 포함률
   - 답변 의도 적합도 게이지
   - 현재는 mock 데이터 + 2초 지연 로딩 구조
6. **커뮤니티 게시판**
   - 공지 / 자유 게시판 분리
   - 글 작성, 수정, 삭제
   - 좋아요 / 북마크 / 댓글 CRUD
   - 로그인 검증 및 API 에러 처리 포함
7. **마이페이지**
   - 사용자 정보 관리
   - 기술 스택 / 관심 분야 수정
   - `/users/me` API 연동
   - 토큰 미보유 시 자동 로그인 페이지 리디렉션
8. **관리자 페이지**
   - 관리자 전용 로그인
   - 다크모드 기반 대시보드
   - 유저 / 카드 / 행동 분석 탭
   - 일반 사용자 인증과 분리된 Zustand 스토어 사용

## 🧩 기술 스택
### ⚛️ Frontend
- React 19
- TypeScript
- React Router v7
- Zustand (상태 관리 + persist)
- Tailwind CSS v4
- Radix UI
- Lucide Icons

### 📦 Data & Utils
- Axios (공용 API 클라이언트)
- xlsx (데이터 처리)

### ⚙️ Dev Environment
- Vite
- @vitejs/plugin-react
- @tailwindcss/vite
- 경로 별칭 `@` 설정 (Vite + TS 통합)
- Hanna 폰트를 `font-air`로 등록

---

## 🏗️ 아키텍처 & 구현 방식
### 🔀 라우팅
- 전체 페이지 lazy loading
- React.Suspense 기반 로딩 처리
- 사용자 / 관리자 뷰를 하나의 라우팅 테이블에서 관리

### 🔐 인증 & 상태 관리
- Zustand + persist 미들웨어 사용
- 사용자 / 관리자 인증 스토어 분리
- Header 및 보호 페이지에서 로그인 상태 구독

### 🌐 API 구조
- 공용 Axios 인스턴스
- 토큰 자동 삽입
- 401 공통 처리
- 도메인별 API 래퍼 (auth, user, post, comment)

### 🎥 미디어 처리
- `useMediaRecorder` 커스텀 훅
- 카메라 / 마이크 제어 로직 분리
- 녹화 에러 및 타이머 상태를 훅 내부에서 관리

### 💾 로컬 상태 유지
- 북마크, 면접 세션 등은 localStorage 활용
- 새로고침 후에도 사용자 맥락 유지

## 🔧 환경 설정
- 기본적으로 `/api` 요청은 `localhost:8080` 백엔드로 프록시됩니다.
- 백엔드 포트가 다를 경우 `.env`에 아래 설정을 추가해 API 경로를 변경할 수 있습니다.

```bash
VITE_API_BASE_URL=http://localhost:포트번호
```
